# Server Owner Setup Guide

## Overview

You host modpack files on any HTTPS web server. Players point the launcher at your `manifest.json` URL.

## Requirements

- Any web server capable of serving static files over HTTPS
- PowerShell (for manifest generation)

## Directory Structure

```
https://mods.example.com/valheim/
├── manifest.json                  # Generated by build-manifest.ps1
├── changelog.txt                  # Optional changelog
├── launcher/
│   └── latest.json                # Optional: launcher self-update info
└── files/
    ├── BepInEx/
    │   ├── plugins/
    │   │   ├── MyMod.dll
    │   │   └── AnotherMod.dll
    │   └── config/
    │       └── MyMod.cfg
    ├── doorstop_config.ini
    └── winhttp.dll
```

## Generating a Manifest

Use the included PowerShell script:

```powershell
.\tools\ManifestBuilder\build-manifest.ps1 `
    -SourceFolder "C:\ModpackFiles" `
    -BaseUrl "https://mods.example.com/valheim/files" `
    -ModpackId "clickcs-server" `
    -ModpackName "ClickCS Valheim Server" `
    -ModpackVersion "1.2.0" `
    -OutputFile "C:\WebRoot\valheim\manifest.json"
```

This will:
1. Scan all files in `SourceFolder`
2. Compute SHA256 hashes for each
3. Generate download URLs based on `BaseUrl`
4. Output `manifest.json`

## Manifest Format

See `docs/MANIFEST_FORMAT.md` for the complete schema.

## Updating the Modpack

1. Update mod files in your source folder
2. Re-run `build-manifest.ps1` with a new `ModpackVersion`
3. Upload both the manifest and changed files to your web server
4. Players click **Check for Updates** in the launcher

## Launcher Self-Update (Optional)

Create `launcher/latest.json`:

```json
{
    "version": "1.1.0",
    "download_url": "https://mods.example.com/valheim/launcher/ClickCS-Valheim-Launcher-1.1.0.exe",
    "sha256": "<sha256 hash of the exe>",
    "changelog_url": "https://mods.example.com/valheim/launcher/changelog.txt",
    "mandatory": false
}
```

Players configure `Launcher Update URL` in Settings to point at this file.

## Web Server Configuration

### Nginx
```nginx
location /valheim/ {
    root /var/www/mods;
    autoindex off;
    add_header Cache-Control "no-cache";
    add_header Access-Control-Allow-Origin "*";
}
```

### IIS (web.config)
```xml
<configuration>
  <system.webServer>
    <staticContent>
      <mimeMap fileExtension=".json" mimeType="application/json" />
      <mimeMap fileExtension=".dll" mimeType="application/octet-stream" />
      <mimeMap fileExtension=".cfg" mimeType="text/plain" />
    </staticContent>
    <httpProtocol>
      <customHeaders>
        <add name="Cache-Control" value="no-cache" />
      </customHeaders>
    </httpProtocol>
  </system.webServer>
</configuration>
```

## Security Notes

- **Always use HTTPS** in production
- The launcher rejects HTTP by default (opt-in only for LAN testing)
- Never put server passwords in the manifest
- All file hashes are verified before installation on the client
- The manifest builder only reads files; it never modifies them
